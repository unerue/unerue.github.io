Travelling Salesman Problem#

외판원 문제(travelling salesman problem, TSP)는 조합 최적화(combinatorial optimization)에서 대표적인 문제로 NP-hard에 속하며 계산 복잡도 이론에서 해를 구하기 어려운 문제입니다. 이 문제는 여러 도시가 존재하고 한 도시에서 다른 도시로 이동하는 비용이 주어졌을 때, 모든 도시를 한 번씩만 방문하고 처음 출발했던 도시로 되돌아오는 총비용 또는 총거리를 최소화하는 경로를 결정하는 것입니다. 이러한 문제를 대칭 외판원 문제(symmetric travelling salesman problem)라고도 하며 그림 1과 같이 대칭 행렬을 뜻합니다. 예를 들면, 도시 간 거리를 단위 킬로미터(km)라하고 행을 출발도시 열을 도착도시라고 정의하면 4번 도시에서 7번 도시까지 거리는 11km입니다.


그림 1 대칭 행렬(symmetric matrix)

외판원 문제는 도시 수가 많아지면 급격하게 어려워지는데 $n$개의 도시가 있을 때 실행 가능한 경로의 수는 출발 도시를 고려하여 $(n-1)!/2$개입니다. $2$로 나눈 이유는 모든 경로의 역 경로는 동일한 해이기 때문입니다. 그래서 10개의 도시가 있는 외판원 문제는 적어도 $200,000$개의 실행가능한 해를 고려해야 합니다. $20$개의 도시는 약 $10^{16}$개, $50$개의 도시는 약 $10^{62}$개입니다. 다음 그림 2는 그림 1의 거리 행렬을 사용하여 경로와 해당 경로의 역은 같은 목적함수 값을 보여줍니다.


그림 2 역경로의 총 거리

외판원 문제는 다양한 곳에서 활용할 수 있는데, 예를 들면 물류 센터에서 각 소매점으로 상품을 배분할 때 트럭이 방문해야할 최단 경로를 결정해야하는 문제나 많은 부품과 칩이 연결된 회로 기판을 제조하는 회사에서 회로 기판에 구멍을 뚫기 위해 구멍을 뚫는 순서를 효율적으로 정할 수 있습니다.

외판원 문제는 다양한 방법으로 해를 구할 수가 있는데, 다음은 혼합 정수 계획법(mixed integer programming)으로 정식화하여 파이썬과 최적화 라이브러리인 PuLP를 사용해서 외판원 문제의 최적해를 구해보겠습니다.

수리 모형#

외판원 문제를 정식화하면 $n$개의 도시가 있을 때 인덱스 $i$와 $j$는 $n$개 만큼있으며 파리미터는 거리 $c_{ij}$, 의사결정변수로 $x_{ij}$와 $u_{i}$가 있으며 Miller-Tucker-Zemlin (MTZ) 공식은 다음과 같습니다.

\[\begin{split}x_{ij} = \begin{cases} 1, \; \text{if the route goes from city $i$ to city $j$,}\\ 0, \; \text{otherwise} \end{cases}\end{split}\]
\[u_{i} = \text{order city $i$ is visited}\]

위의 의사결정변수와 파라미터를 이용하여 총 거리의 합을 최소화하는 정수 계획법의 목적함수는 식(1)과 같습니다.

\[\begin{equation} \text{minimize }\sum_{i=1}^{n} \sum_{j \ne i,j=1}^{n} c_{ij}x_{ij} \end{equation}\]

제약식은 다음과 같습니다. 제약식(2)과 (3)은 각 도시 $j$로 들어오는 도시는 1개가 되어야만 하고 각 도시 $i$에서 출발하는 도시는 1개가 되어야 한다는 걸 나타냅니다.

\[\begin{equation} \sum_{i=1, i \ne j}^{n} x_{ij} = 1, \qquad j = 1, \dots, n, \end{equation}\]
\[\begin{equation} \sum_{j=1, j \ne i}^{n} x_{ij} = 1, \qquad i = 1, \dots, n, \end{equation}\]

제약식(3)은 부등식 제약조건(inequality constraints)으로 MTZ 공식입니다. $x_{ij}=1$일 때, $(i,j)$에 대한 두 도시 사이는 순서는 $u_{j} \ge u_{i}$를 뜻합니다. 제약식(4)와 (5)는 의사결정변수 $x_{ij}$는 $0$과 $1$ 값만 가질 수 있는 이진변수이고, $u_{i}$는 $0$보다 크거나 같고 $n-1$보다 작거나 같은 값을 가질 수 있는 변수를 나타냅니다.

\[\begin{equation} u_{i} - u_{j} + nx_{ij} \le n-1 \qquad 2 \le i \ne j \le n, \end{equation}\]
\[\begin{equation} x_{ij} \in {0,1} \qquad i,j = 1, \dots, n, \end{equation}\]
\[\begin{equation} 0 \le u_{i} \le n-1 \qquad i = 2, \dots, n. \end{equation}\]

외판원 문제 무작위 생성#

외판원 문제의 최적해를 구하기 전, 무작위로 각 도시의 좌표를 생성하고 두 도시 사이의 거리를 유클리드 거리(euclidean distance)로 계산하여 대칭 행렬을 생성하기 위해 클래스로 구현해보겠습니다. 해당 클래스는 앞으로 대표적인 메타휴리스틱스를 구현하기 위해 기본 라이브러리로 사용될 것입니다.

    """

        self.y = y

혼합정수계획모형#

이제 파이썬을 활용하여 혼합정수계획모형(mixed-integer programming)을 구현하고 최적해를 구하겠습니다. 파이썬을 활용한 경영과학(MS/OR)은 추후 기초부터 포스팅을 할 예정입니다.

from pulp import *

# Initialize travelling salesman problem
prob = LpProblem('Travelling Salesman', LpMinimize)

n = len(cities)
indexs = [(i, j) for i in range(n) for j in range(n) if i != j]

# Creating decision variables
x = LpVariable.dicts('x', indexs, cat='Binary')
u = LpVariable.dicts('u', list(range(n)), lowBound=0, upBound=n-1, cat='Continuous')

# Objective function
prob += lpSum([cities[i][j] * x[(i,j)] for i, j in indexs])

# Constraints
for i in range(n):
    prob += lpSum([x[(i,j)] for j in range(n) if i != j]) == 1
    
for j in range(n):
    prob += lpSum([x[(i,j)] for i in range(n) if i != j]) == 1
    
for i in range(1, n):
    for j in range(1, n):
        if i != j:
            prob += u[i] - u[j] + n * x[(i,j)] <= n - 1

# Solve problem
prob.solve()
print(value(prob.objective))

for i in prob.variables():
    if i.name[0] == 'u':
        print(i.name, '=', i.varValue)
    elif i.varValue != 0:
        print(i.name, '=', i.varValue)

완전 탐색#

문제를 푸는 데 있어서 가장 쉽고 간단한 방법은 모든 해를 일일이 탐색하는 것입니다. 이것을 브루트 포스 탐색(brute-force search) 또는 완전 탐색(exhaustive search)이라고 합니다. 명칭 그대로 모든 해를 탐색하기 때문에 계산소요시간은 지수적으로 발생합니다. 현실 문제 대부분은 영원히 끝나지 않는 반복을 볼 수가 있을 겁니다. 외판원 문제에 완전 탐색을 적용하여 최적해를 구해보도록 하겠습니다. 문제의 크기가 작기 때문에 가능합니다.

import numpy as np
from itertools import permutations


class BruteForce:
    """Brute Force Search
  
    Parameters
    ----------
    start : start city
    cities : distance matrix
        
    Returns
    -------
    self.best_distance : 
    """
    def __init__(self, start, cities):
        self.start = start
        self.cities = cities
        self.num_cities = len(self.cities)
        
    def _get_distance(self, route):
        return np.sum([self.cities[i, j] for i, j in zip(route, route[1:] + [0])])
        
    def solve(self):
        self.best_distance = np.inf
        self.best_solution = None
        
        for route in permutations(range(self.num_cities)):
            current_route = list(route)
            if current_route[0] == self.start:
                current_distance = self._get_distance(current_route)

                if current_distance <= self.best_distance:
                    self.best_distance = current_distance
                    self.best_route = current_route
                    
        return self.best_distance

brute_force = BruteForce(0, cities)
brute_force.solve()

최근접 이웃 휴리스틱#

외판원 문제는 휴리스틱으로 최적해는 아니지만 만족할만한 근사해를 구할 수 있습니다. 다음은 최근접 이웃 휴리스틱 절차를 설명하고 파이썬으로 구현해보도록 하겠습니다.

  • STEP 1: Initialize
    1. 방문한 도시(visited cities)와 앞으로 방문할 도시(cities to visit)의 리스트를 생성한다.
    2. 출발도시는 1번 도시로 선택한다.
  • STEP 2:
    1. 방문한 도시 리스트에 1번 도시를 넣어두고, 방문할 도시 리스트에는 1번 도시를 삭제한다.
    2. 1번 도시를 기준으로 방문할 도시 $[2, 3, 4, 5, 6, 7, 8, 9, 10]$ 중에서 1번 도시와 거리가 가장 가까운 도시를 선택한다.
    3. 방문한 도시 리스트에서 1번 도시 다음 위치에 STEP 2.2 선택한 도시를 넣고, 방문할 도시에서 해당 도시를 삭제한다.
    4. 방문한 도시 리스트 가장 최근에 넣어진 도시를 기준으로 STEP 2.2 과정을 반복한다.
  • STEP 3: 방문할 도시의 리스트 내 모든 도시가 없어질 때까지 반복한다.
class NearestNeighbor:
    """Nearest Neighbor Heuristic
  
    Parameters
    ----------
    start : start city
    cities : distance matrix
        
    Returns
    -------
    self.best_distance : 
    """
    def __init__(self, start, cities):
        self.start = start
        self.cities = cities
        self.visited_cities = [self.start]
        self.cities_to_visit = list(range(len(self.cities)))
        self.cities_to_visit.remove(self.start)
        
    def _get_distance(self, route):
        return sum([self.cities[i, j] for i, j in zip(route, route[1:] + [route[0]])])
    
    def solve(self):
        while self.cities_to_visit:
            nearest_city = min([(self.cities[self.start][j], j) for j in self.cities_to_visit], key=lambda x: x[0])
            self.start = nearest_city[1]
            self.cities_to_visit.remove(self.start)
            self.visited_cities.append(self.start)
        self.best_distance = self._get_distance(self.visited_cities)
        return self.best_distance

nearest_neighbor = NearestNeighbor(0, cities)
nearest_neighbor.solve()

결론#

조합 최적화의 대표적인 문제인 외판원 문제에 대해서 알아보았습니다. 또한, 외판원 문제의 최적해를 도출하기 위해서 혼합 정수 계획법과 완전 탐색을 적용하였고, 합리적인 시간에 근사한 해를 구할 수 있는 휴리스틱한 방법인 최근접 이웃을 파이썬으로 구현했습니다. 외판원 문제는 다양한 알고리즘으로 해를 구할 수 있습니다. 그 중 2016년 Google Brain에서 강화학습(reinforcement learning)으로 조합 최적화 문제를 다루는 논문을 발표했습니다. 논문의 제목은 Neural Combinatorial Optimization with Reinforcement Learning으로 앞으로 메타휴리스틱스 중에서 대표적인 알고리즘인 시뮬레이티드 어닐링, 타부 서치, 유전 알고리즘, 입자 군집 최적화, 개미 군집 최적화 등을 이용해서 외판원 문제를 다뤄보고 구글에서 적용한 강화학습을 파이토치(pytorch)로 구현해보도록 하겠습니다.

해당 포스트는 지속적으로 업데이트 할 예정이니 오타, 틀린부분, 파이써닉(pythonic)하지 못한 코드가 있을 경우 지적해주시면 수정하도록 하겠습니다.
